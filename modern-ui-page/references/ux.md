# UX Checks for Page-Level UI

## Purpose Clarity

The page should answer these quickly:
- What is this?
- Why should I care?
- What should I do next?

If the user must study the interface to understand the core value, the page is not done.

## Information Hierarchy

- Lead with the most important message
- Group related content into clear sections
- Use spacing, typography, and layout to support scanability
- Avoid equal visual weight everywhere

## Primary Action

- The main CTA should be easy to find
- Competing actions should not overpower the primary action
- Repeating the CTA is fine when the page is long, but do it intentionally

## Content Density

- Marketing pages usually benefit from focus and breathing room
- Dashboard pages can be denser, but the layout still needs visual order
- Do not confuse crowding with sophistication

## States and Feedback

Where relevant, make space for:
- loading states
- empty states
- error states
- success or confirmation states

Even if not fully implemented, the page structure should not make those states awkward later.

## Responsive Behavior

- The mobile layout should preserve hierarchy, not just stack everything blindly
- Important actions should remain visible or easy to reach
- Avoid layouts that only feel balanced on wide screens

## Common UX Failure Modes

- Beautiful page with no obvious next step
- Overdesigned hero that pushes useful content too far down
- Sections that repeat the same message without adding value
- Dense pages with weak grouping and no visual rhythm
- Mobile version that feels like a collapsed desktop page

## Final Check

Before handoff, confirm:
- the page communicates value quickly
- the next action is obvious
- the reading path feels intentional
- visual flourish is supporting the experience, not replacing it
